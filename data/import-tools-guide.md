# 导入工具数据说明

这份说明配合：

- `scripts/import-tools-to-submissions.ts`
- `data/tools-import.sample.json`

一起使用。

## 这一步的作用

批量导入时，先把数据写进 `submission`，再通过后台审核进入 `tool`。  
这样更安全，也更适合你现在“收口阶段 → 试运营准备阶段”的工作流。

---

## JSON 顶层格式

导入文件必须是 **数组**，每一项代表一条待导入工具数据。

```json
[
  {
    "name": "工具名称",
    "website": "https://example.com",
    "description": "一句话简介",
    "category": "主分类",
    "tags": ["标签1", "标签2"],
    "contact": "",
    "reason": "补充说明"
  }
]
```

---

## 字段说明

### 1. `name`
必填。  
工具正式名称，尽量统一用官方主名称。

建议：
- 用官方名称
- 不要带营销口号
- 不要把分类词硬拼进名称里

正确示例：
- `Kimi`
- `Claude`
- `Runway`

不建议：
- `Kimi AI 智能长文本写作助手`
- `最好用的视频生成工具 Runway`

---

### 2. `website`
必填。  
必须是完整的 `http` 或 `https` 地址。

正确示例：
- `https://www.perplexity.ai`
- `https://runwayml.com`

不建议：
- `runwayml.com`
- `www.perplexity.ai`

脚本会自动去掉结尾多余的 `/`。

---

### 3. `description`
必填。  
这是**前台一句话简介**，不要写成长文。

建议写法：
- 一句话说明“这是什么工具”
- 语气客观
- 长度控制在一句话内

正确示例：
- `一款结合搜索与问答体验的 AI 搜索工具。`
- `一款面向视频生成、视频编辑和创意制作的 AI 工具。`

不建议：
- 写成功能列表堆砌
- 写成广告语
- 写后台审核备注

---

### 4. `category`
必填。  
只填写 **一个主分类**。

正确示例：
- `聊天助手`
- `视频生成`
- `AI搜索`
- `AI写作`

不建议：
- `聊天助手 / 写作助手`
- `AI搜索, 问答工具`
- `分类`
- `未分类`

脚本会拦截组合分类值。

---

### 5. `tags`
选填。  
可以写成数组，也可以写成逗号分隔字符串。

两种都可以：

```json
"tags": ["AI写作", "中文支持", "长文本"]
```

```json
"tags": "AI写作, 中文支持, 长文本"
```

规则：
- 建议 3 到 8 个
- 尽量少而准
- 避免重复
- 避免超长标签

---

### 6. `contact`
选填。  
可填写投稿来源、邮箱、联系人标记等。  
没有就留空字符串。

---

### 7. `reason`
选填。  
这是**后台补充说明 / 导入备注**，不是前台简介。

适合写：
- 为什么要收录
- 归类依据
- 去重说明
- 需要审核时注意的点

不建议：
- 重复 description
- 写成前台介绍文案

---

## 当前推荐写法

### 推荐最小模板

```json
{
  "name": "Perplexity",
  "website": "https://www.perplexity.ai",
  "description": "一款结合搜索与问答体验的 AI 搜索工具。",
  "category": "AI搜索",
  "tags": ["AI搜索", "问答", "信息检索"],
  "contact": "",
  "reason": "适合 AI 搜索分类。"
}
```

---

## 常见错误

### 1. 分类写成多个
错误：

```json
"category": "聊天助手 / AI写作"
```

应改为：

```json
"category": "聊天助手"
```

---

### 2. description 写太长
错误：

```json
"description": "这是一款非常强大的 AI 工具，拥有很多能力，适合很多场景，还能……"
```

应改为：

```json
"description": "一款支持长文本理解与写作辅助的 AI 助手。"
```

---

### 3. 标签太多
不建议一次塞十几个标签。  
目录站更适合少而准。

---

### 4. website 不完整
错误：

```json
"website": "runwayml.com"
```

应改为：

```json
"website": "https://runwayml.com"
```

---

## 导入建议流程

1. 先整理 JSON
2. 用样例模板自查字段
3. 在测试库运行导入脚本
4. 进入 `/admin/submissions` 审核
5. 审核通过后再进入前台

---

## 当前导入策略

继续沿用这条路线：

- **不直接写 `tool`**
- **先写 `submission`**
- **再走后台审核**

这是目前最稳的方案。
